#### N0018 r0015 — Multi-Vault 설계 결정

`@vault:` 링크 네임스페이스 확정:

| 문법 | 의미 |
|------|------|
| `@vault:global:N####` | 절대 참조 (`~/.elendirna/`) |
| `@vault:<alias>:N####` | config named vault |
| `@vault:local:N####` | 상대 참조 (링크 지양, 파라미터 전용) |
| `N####` | 동일 vault 내 참조 (현행 유지) |

**cwd = 인텐트 신호 검증:** 현행 탐색 우선순위(cwd → 상위 → `~/.elendirna/` 폴백)가 이미 올바름. `--vault` 파라미터는 예외 탈출구 위상. 코드 변경 없음.

#### N0029 r0002 — FAQ "그냥 Git 쓰면 안 되나요?" 추가

설계가 진행될수록 Git과 유사한 구조(revision/baseline, index, sync log 등)가 나타남. 이는 "변경 이력이 있는 분산 저장소" 문제의 자연스러운 수렴.

**Elendirna ≠ Git인 핵심 이유:**
- **버전 단위:** Git은 파일, Elendirna는 *아이디어* first-class
- **delta 의미론:** Git diff는 줄 단위(자동), delta는 의미적("왜 바뀌었나") 서술(수동)
- **AI-native 레이어:** sync_record, bundle, MCP 통합 — 에이전트 능동 탐색 워크플로

→ see N0029 r0002
