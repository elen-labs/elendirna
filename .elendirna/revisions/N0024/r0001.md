## Delta

# 사용 시나리오: Slack/Messaging App Integration (Channel as Vault)

> **상태**: 제안 단계 — 협업 메신저 내에서 지식의 결정화(Crystallization)를 지원하는 Elendirna 활용 예시
> **핵심 가치**: "채팅의 휘발성을 Elendirna의 구조적 보존성으로 극복"

---

## 1. 배경: Slack 채널의 지식 증발 문제

많은 협업이 Slack에서 일어나지만, 중요한 결정과 맥락은 수천 개의 메시지 뒤로 밀려나 잊혀집니다. `elendirna`가 Slack 앱으로 통합되면, 채널 자체가 하나의 **Project Vault**가 되어 실시간으로 지식을 구조화합니다.

---

## 2. 인터랙티브 지식 기록 (Crystallization)

팀원들은 평소처럼 대화하지만, 봇(Elf-Bot)이 결정적인 순간을 감지하거나 사용자의 명시적 요청에 따라 Vault를 업데이트합니다.

### 2-1. 자동 감지 및 제안 (Bot Proactive)
- **철수**: "이번 UI 시안은 다크 모드를 기본으로 가기로 합시다."
- **Elf-Bot**: (📌 이모지 반응 후) "UI 디자인 결정 사항(`N0042`)에 새로운 `revision`을 추가할까요?"
- **철수**: "그래, 추가해 줘."
- **Elf-Bot**: `elf revision add N0042 --delta \"다크 모드를 기본으로 채택\" --author chulsu` 실행 완료.

### 2-2. 명령어를 통한 수동 기록 (User Explicit)
- **영희**: `/elf entry new \"API 인증 방식: OAuth2\"`
- **Elf-Bot**: "새로운 Entry `N0043`이 생성되었습니다. 상세 내용은 채널 스레드에서 작성해 주세요."

---

## 3. 맥락 소환 및 브리핑 (Context Retrieval)

신규 입사자나 프로젝트에 중도 투입된 멤버를 위해 봇이 Vault의 지식 계보를 브리핑합니다.

### 3-1. 프로젝트 타임라인 요약
- **신입 사원**: `/elf bundle status`
- **Elf-Bot**: (bundle chain 분석 후)
  > "이 채널(#project-A-vault)은 3개월 전 `N0001`에서 시작되었습니다. 
  > 초기에는 클라우드 서버 선정을 고민했고(N0012), 
  > 1개월 전 비용 문제로 AWS에서 GCP로 선회했습니다(N0015). 
  > 현재 가장 활발히 논의 중인 주제는 API 설계(N0043)입니다."

---

## 4. 채널 간 지식 공유 (Cross-Vault Link)

서로 다른 채널(Vault) 간의 의존성을 관리합니다.

### 4-1. 전사 정책 참조
- **보안팀 채널**: `#security-policy-vault`
- **개발팀 채널**: `#backend-dev-vault`
- **영희**: `/elf link #security-policy-vault:N0005 #backend-dev-vault:N0043`
- **Elf-Bot**: "백엔드 API 설계가 전사 보안 정책(로그인 보안)과 공식적으로 연결되었습니다."

---

## 5. 기술적 요구사항 및 제안 (Proposal 연계)

이 시나리오를 위해 Elendirna CLI에 추가로 필요한 기능들입니다.

1.  **Remote-Vault Support**: 로컬 파일 시스템이 아닌 클라우드(S3, Git 등)에 저장된 Vault를 읽고 쓸 수 있는 기능.
2.  **Author Mapping**: Slack 사용자 ID를 Elendirna의 `author` 필드와 매핑하는 기능.
3.  **Webhook & Event Stream**: 채팅방의 특정 이벤트를 `elf` 명령어로 변환하는 미들웨어.
4.  **Markdown-to-Slack Formatting**: `elf bundle`의 마크다운 출력을 Slack의 블록 키트(Block Kit) 포맷으로 변환하는 렌더러.

---

## 6. 기대 효과

1.  **온보딩 비용 80% 감소**: 수개월 치의 채팅 로그 대신, 봇이 요약해 주는 `bundle` 체인만 읽으면 즉시 작업 투입 가능.
2.  **의사결정 투명성**: "누가 왜 바꿨는가?"가 명확히 기록되어 팀 내 신뢰도 향상.
3.  **지식 자산화**: 단순한 노이즈(채팅)를 정제된 지식(Vault)으로 자동 변환.
