# Proposal 005: Collaborative Identity and Bot Support

> **상태**: Proposed
> **대상 마일스톤**: v0.4 (협업 지원 단계)
> **관련 시나리오**: → see N0022 (Collaborative Debate)

---

## 1. 배경 및 동기

현재 Elendirna는 단일 사용자와 단일 AI 에이전트 간의 관계를 가정하고 설계되었습니다. 하지만 실제 개발 환경에서는 **하나의 지식 저장소(Vault)를 여러 명이 공유**하거나, **AI 봇이 팀원으로서 채팅방에 참여**하여 기록을 관리하는 요구사항이 발생합니다. 

본 제안서는 Elendirna를 "개인용 기억"에서 "팀의 공유 지능"으로 확장하기 위한 기술적 변화를 정의합니다.

---

## 2. 주요 제안 사항

### 2-A. Identity(작성자) 필드 도입
모든 지식의 생성 및 수정 시점에 작성자를 기록하여 책임성과 맥락을 보존합니다.

- **`manifest.toml` 확장:** `author` 필드(Optional) 추가.
- **`revision` metadata 확장:** 각 delta 파일의 프론트매터에 `author` 필드 추가.
- **`sync.jsonl` 확장:** 세션 로그에 `author` 필드 추가.

### 2-B. CLI 명령어 확장
사용자 식별을 위한 인자를 모든 변경 명령에 추가합니다.

- `elf entry new <title> --author <name>`
- `elf revision add <id> --delta <text> --author <name>`
- `elf sync record --summary <text> --author <name>`

*참고: 설정 파일(`config.toml`)에 `default_author`를 지정하여 매번 입력하는 번거로움을 줄일 수 있습니다.*

### 2-C. 봇(Bot) 지원을 위한 인터페이스 강화
봇이 지식의 촉매제 역할을 수행할 수 있도록 도구를 최적화합니다.

1.  **`elf bundle --include-linked` 개선:** 봇이 여러 노드 간의 논쟁 흐름을 한꺼번에 읽어들여 요약할 수 있도록 출력을 최적화합니다.
2.  **SSE(Server-Sent Events) Transport:** (N0013 OQ-6 연계) 원격 챗봇 플랫폼과의 실시간 통신을 위한 서버 모드를 지원합니다.
3.  **Ambiguity Detection (추후 고려):** 동일한 baseline에서 여러 author의 revision이 발생할 경우, 봇이 이를 "논쟁(Debate)"으로 감지하고 요약을 제안하는 트리거 로직을 제공합니다.

## 4. 기대 효과

- **지식의 투명성**: "왜 이 결정이 내려졌나?" 뿐만 아니라 "누가 이 의견을 냈나?"를 추적 가능.
- **비동기 합의**: 팀원이 서로 다른 시간에 기록을 남겨도 `Base-Delta` 체인으로 자연스럽게 통합.
- **AI 보조**: 봇이 단순한 저장소가 아닌, 팀의 과거 결정을 현재의 논의로 소환하는 지능형 비서로 진화.
