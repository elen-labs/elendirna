## Delta

## --tag → --tags CLI 플래그 개선 proposal (2026-04-14)

현행 `elf entry list` 및 `elf query` 모두 `--tag <TAG>` 단수형만 지원. 복수 태그 필터링 불가.

### 검토 방안

**Option A — 플래그 이름 변경: `--tag` → `--tags`**
- `--tags rust,design` (콤마 구분) 또는 `--tags rust --tags design` (반복 사용) 형태
- 단수형 제거 → 기존 스크립트/alias 호환성 깨짐
- 의미적으로 더 정확 (복수 값 허용임을 이름에서 드러냄)

**Option B — `--tag` + `--tags` 동시 지원 (alias)**
- 단수형 `--tag` 유지 + `--tags` 추가 alias
- 하위 호환성 보존, 점진적 이전 가능
- clap에서 `alias` 또는 별도 `--tags` 정의 후 내부 병합으로 구현

**Option C — `--tag` 반복 허용 (현행 + multi)**
- `--tag rust --tag design` → AND 필터
- 플래그 이름 변경 없이 `Vec<String>`으로 타입 변경만 필요
- 가장 변경 범위 작음, 기존 `--tag foo` 호환 유지

### 권장

Option C를 v0.4에 포함하거나 별도 patch로 처리. 이름 변경은 별도 breaking change 릴리즈에서 Option B(alias 병행)로 진행 권장.

→ see N0004 r0003 (v0.4 스코프 확정 목록)
