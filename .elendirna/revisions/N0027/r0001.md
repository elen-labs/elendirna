# Proposal 006: Explicit Status Management and Workflow Control

## 1. 배경 (Context)
- 현재 Elendirna는 `draft`, `stable`, `archived`의 상태를 지원하지만, 이를 명시적으로 변경할 수 있는 CLI 명령어가 부재함.
- 사용자는 `manifest.toml`을 직접 수정해야 하며, 이는 "모든 조작은 CLI를 통해 강제한다"는 설계 원칙에 위배됨.
- 또한, AI 에이전트와 사람 사이에서 언제 지식을 `stable`로 격상시킬지에 대한 구체적인 합의 기준이 문서화되지 않음.

## 2. 제안 사항 (Proposed Changes)

### 2.1 신규 명령어: `elf entry status <id> <status>`
- **사용법:** `elf entry status N0042 stable`
- **동작:**
  - `manifest.toml`의 `status` 필드를 업데이트.
  - `updated` 타임스탬프 갱신.
  - `sync.jsonl`에 상태 변경 이벤트 기록 (`status.changed`).
- **검증:** 유효하지 않은 상태 값 입력 시 에러 반환.

### 2.2 상태 전환 기준 (Workflow Criteria)

| 전환 경로 | 트리거 (Trigger) | 의미 (Semantics) |
|-----------|------------------|-------------------|
| **Draft → Stable** | 논의 완료 및 합의 (Consensus) | "신뢰할 수 있는 지식(SSoT)"으로 승격. 다른 엔트리의 기반이 됨. |
| **Stable → Archived** | 대체(Superseded) 또는 폐기 | "역사적 기록"으로 전환. 기본 검색 노이즈에서 제외. |
| **Draft → Archived** | 기각(Rejected) 또는 중단 | 실패한 실험이나 폐기된 아이디어의 기록 보존. |

## 3. 에이전트 가이드라인
- 에이전트는 작업 세션 종료 시, 충분히 검증된 `draft` 엔트리를 `stable`로 전환할 것을 제안해야 함.
- `bundle`을 통해 컨텍스트를 구성할 때, `stable` 엔트리는 '절대적 지침'으로, `draft` 엔트리는 '검토 중인 의견'으로 구분하여 처리함.

## 4. 향후 과제
- `elf list`의 기본 필터링 정책 결정 (예: `archived`는 기본적으로 제외).
- 상태 전환 시 자동으로 관련 링크들의 무결성을 재검사하는 기능 추가.
