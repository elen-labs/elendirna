# 사용 시나리오: 다인 협업과 봇의 중재 (Collaborative Debate)

> **상태**: 제안 단계 — 다중 사용자 환경에서의 Elendirna 활용 예시
> **핵심 가치**: "누가, 왜 이 결정을 바꿨는가?"에 대한 팀의 투명한 기록

---

## 배경: 신규 프로젝트 DB 선정 (PostgreSQL vs MongoDB)

개발팀(철수, 영희)과 AI 에이전트(Elf-Bot)가 채팅방에서 프로젝트 아키텍처를 결정하고 있습니다. Elendirna는 이 과정의 결정적 순간들을 기록하는 저장소가 됩니다.

---

## Phase 1: 초기 제안 (철수 - RDBMS 지지)

철수는 데이터의 일관성이 가장 중요하다고 판단하여 첫 번째 제안을 등록합니다.

### 1-1. Entry 생성 (철수)
```sh
$ elf entry new \"DB 선정: PostgreSQL\" --tags architecture,db --author chulsu
```
> *(가정: `--author` 필드는 다인 협업을 위한 확장 기능)*

**철수의 제안 내용 (`note.md`):**
- 엄격한 스키마를 통한 데이터 품질 보장
- 트랜잭션의 ACID 특성 필요
- 복잡한 JOIN 쿼리 효율성

---

## Phase 2: 반론과 대안 (영희 - NoSQL 지지)

영희는 개발 속도와 유연한 스키마가 더 중요하다고 생각하여 반대 의견을 제시합니다. 영희는 철수의 제안을 `baseline`으로 삼아 자신의 논리를 전개합니다.

### 2-1. 파생 Entry 생성 (영희)
```sh
$ elf entry new \"DB 선정: MongoDB (NoSQL)\" --baseline N0031 --tags architecture,db --author younghee
```

**영희의 반론 내용 (`note.md`):**
- 초기 개발 단계에서의 잦은 스키마 변경 대응
- JSON 포맷의 데이터 처리가 프런트엔드와 궁합이 좋음
- 수평적 확장의 용이성

---

## Phase 3: 봇의 중재 (Elf-Bot - 퍼실리테이터)

채팅방의 **Elf-Bot**은 두 사람의 논쟁이 평행선을 달릴 때, 지금까지의 논점을 정리하여 합의를 유도합니다.

### 3-1. 논쟁 요약 (Bot)
봇은 `elf bundle` 명령어를 통해 두 아이디어의 계보와 차이점을 분석합니다.

```sh
$ elf bundle N0031 --include-linked
```

**봇의 요약 보고 (채팅방 메시지):**
> "철수님(N0031)은 **데이터 무결성**을, 영희님(N0032)은 **유연성**을 강조하고 계십니다. 
> 현재 합의되지 않은 지점은 '향후 6개월 내 데이터 스키마의 안정성'입니다. 
> 과거 프로젝트(N0012)에서 MongoDB를 썼을 때 JOIN 성능 문제가 있었던 점을 고려하면 어떨까요?"

---

## Phase 4: 합의 및 최종 기록 (팀의 결정)

긴 논의 끝에 팀은 PostgreSQL을 쓰되, 일부 메타데이터는 JSONB 타입으로 저장하기로 합의합니다. 봇은 이 최종 결론을 기록합니다.

### 4-1. 최종 Revision 추가 (Bot)
```sh
$ elf revision add N0031 --delta \"합의 완료: PostgreSQL 채택 (JSONB 활용). 스키마 유연성은 JSONB로 보완하며, 핵심 데이터는 RDBMS의 ACID를 유지하기로 함.\" --author elf-bot
```

### 4-2. 세션 종료 기록
```sh
$ elf sync record --summary \"DB 선정 토론 세션 종료. 철수(RDBMS) vs 영희(NoSQL) 논의 후 PostgreSQL + JSONB 혼합 전략으로 최종 결정.\"
```

## 이 시나리오가 보여주는 다인 협업의 이점

1.  **책임성 (Accountability)**: `author` 필드를 통해 각 결정의 주체를 명확히 합니다.
2.  **맥락 보존 (Context Preservation)**: 단순히 결과만 남는 게 아니라, 어떤 논쟁을 거쳐 합의에 이르렀는지(Revision Chain)가 남습니다.
3.  **지능형 보좌 (Agent Assistance)**: AI가 과거의 유사한 결정 사례(N0012)를 소환하여 현재의 논쟁에 실질적인 도움을 줍니다.
4.  **Bot as a Member**: 봇이 단순한 도구가 아니라 기록과 요약을 전담하는 '팀원'으로 참여합니다.
