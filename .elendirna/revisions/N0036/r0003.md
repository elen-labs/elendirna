## Delta

## 핵심 통찰 및 의사결정

### 통찰 1: "도구가 스스로를 설명한다"
MCP tool description과 server.instructions에 트리거 조건/워크플로를 내장하면
CLAUDE.md 없이도 에이전트가 올바른 워크플로를 자동으로 이해한다.
이는 "설정 파일 의존 제거" 방향의 중요한 진전이다.

### 통찰 2: 컨텍스트 예산 자기결정권
bundle --depth/--since 는 에이전트가 컨텍스트 소비량을 직접 제어할 수 있게 한다.
이전에는 bundle이 모든 linked entry를 무조건 full body로 가져왔으나,
이제 depth=0으로 최소화하거나 since=timestamp로 최신 delta만 가져올 수 있다.

### 통찰 3: worktree 기반 개발 파이프라인
worktree(elastic-williams)에서 개발 → PR 생성 → main merge → publish 흐름이 확립되었다.
시스템에 설치된 elf 버전과 worktree 내 빌드된 버전이 다를 수 있음을 항상 인지해야 한다.

### 의사결정: elf serve UX 전환
--mcp 없이 elf serve 호출 시 에러를 내는 대신 config snippet을 안내한다.
"에러 메시지 → 도움이 되는 안내"로의 전환. 바이너리 경로/vault 경로를 자동 삽입.

### 의사결정: alias 예약어
global / local 은 alias 등록 불가로 결정. 혼동 방지.
