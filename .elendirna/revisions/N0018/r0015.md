## Multi-Vault 설계 결정 (2026-04-14 세션)

### `@vault:` 네임스페이스 확정

**링크 문법:**
- `@vault:global:N####` — 절대 참조. `~/.elendirna/`를 항상 가리킴. 안정적.
- `@vault:<alias>:N####` — config에서 정의된 named vault 참조.
- `@vault:local:N####` — 상대 참조. 링크 문법으로는 불안정하므로 지양. 파라미터 전용.
- 동일 vault 내 참조 — `N####` 그대로 유지.

**CLI/MCP 파라미터:**
- `--vault=local`, `--vault=global` 예약 키워드로 선점.
- `--vault=<alias>` — config 정의 named vault.
- `local`/`global`은 파라미터 컨텍스트에서는 의미가 안정적 (링크와 달리).

### cwd = 인텐트 신호 (설계 검증)

**관찰:** 사용자는 local 지식을 다루려면 프로젝트 디렉터리에서 에이전트를 열고, global 지식이 필요하면 `%USERPROFILE%`에서 에이전트를 연다. cwd 선택 자체가 이미 인텐트를 표현한다.

**결론:** 현행 탐색 우선순위(cwd → 상위 → `~/.elendirna/` 폴백)가 이 워크플로를 이미 정확하게 반영. default vault 결정 규칙은 이미 해결된 상태였음.

**`--vault` 파라미터의 위상:** 예외 탈출구. 로컬 세션 중 글로벌 entry 참조 등 드문 케이스에만 사용.

### 버전 영향
- cwd 인텐트 관찰 → 코드 변경 없음, 버전 불필요 (현행 동작 검증)
- `@vault:` 링크 문법 파서, `--vault` CLI/MCP 파라미터 → v0.4 구현 항목
