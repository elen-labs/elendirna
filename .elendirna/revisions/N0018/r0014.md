## Dogfooding 딜레마 후속 관찰 (2026-04-13)

r0013의 MCP 우회 현상이 **dogfooding vault 특유의 문제**임을 확인.

**재현 조건 비교:**
- 빈 vault + MCP 서버만 → 에이전트가 MCP 도구를 정상 사용
- 코드베이스 내 .elendirna/ 존재 시 → 에이전트가 sync.jsonl 직접 읽기를 선호

**원인:** 에이전트는 프로젝트 디렉터리 내 파일을 "소스 코드의 일부"로 인식. file browsing이 MCP보다 저항이 낮은 경로이므로, 보이는 파일을 도구로 우회해서 읽을 동기가 없음.

**일반 사용자에겐 발생하지 않는 이유:** vault가 ~/.elendirna/(글로벌)에 위치하여 프로젝트 코드에서 보이지 않음 → MCP만이 유일한 접근 경로.

**아이러니:** dogfooding을 위해 vault를 코드베이스에 커밋해야 하지만, 그 행위 자체가 MCP 우회를 유발. .gitignore에 넣으면 문제 해결되지만 dogfooding 이력 관리 불가.

→ AGENTS.md / GEMINI.md 등의 instruction 파일이 이 특수 상황에서 더욱 필요한 이유.
