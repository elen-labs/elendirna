## 실증 관찰: 에이전트의 MCP 우회 현상 (2026-04-13)

Gemini가 동일한 "vault 유지보수 이력" 질문에 MCP tool 대신 `.elendirna/sync.jsonl`을 직접 읽는 현상 확인.

**원인 분석:**
- `sync.jsonl`이 프로젝트 루트 `.elendirna/`에 노출되어 있어 file browsing으로 즉시 발견 가능
- Claude는 CLAUDE.md의 명시적 강제("반드시 MCP tool 사용")로 억제되지만, Gemini에는 동등한 instruction이 없음
- MCP 도구 설명에 "파일 직접 접근 대신 이 도구를 사용해야 하는 이유"가 부재 → 에이전트 입장에서 파일 읽기가 합리적 대안

**구조적 문제:**
직접 파일 접근: 쉽고, 도구 불필요, 즉각적
MCP 도구 사용: 설득력 있는 이유가 있어야 선택

§3.5(MCP 도구 설명 규칙)의 현실적 한계: 도구 설명만으로는 file browsing 능력이 있는 에이전트를 충분히 유도하지 못함.

**개선 방향:**
1. MCP 도구 설명에 "직접 파일 읽기 대신 사용" 명시 + 이유(인덱스 정합성 보장)
2. `AGENTS.md` 도입 — Gemini, Codex 등 multi-agent 표준으로 부상 중인 파일명. N0019 액션 아이템으로 이관 예정.
