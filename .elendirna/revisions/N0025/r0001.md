# Feature: Multi-dimensional Knowledge Visualization Strategy

> **상태**: 제안 단계 — Elendirna의 구조화된 데이터(Vault)를 입체적으로 시각화하는 전략
> **핵심 가치**: "복잡한 지식의 연결과 시간적 변화를 한눈에 통찰"
> **관련 시나리오**: → see N0022 (Debate), → see N0024 (Slack Integration)

---

## 1. 배경: 지식의 텍스트 한계 극복

Vault에 쌓이는 수백 개의 Entry와 Revision은 텍스트로만 존재할 때 전체적인 지형도를 파악하기 어렵습니다. 시각화는 다음 질문들에 대한 답을 시각적으로 제공합니다.
- "지금 우리 프로젝트에서 가장 뜨거운 논쟁 주제는 무엇인가?"
- "이 아이디어는 어떤 단계를 거쳐 현재의 모습이 되었는가?"
- "우리 팀의 지식은 어떤 분야에 편중되어 있는가?"

---

## 2. 3대 시각화 차원 (Three Dimensions)

### A. 지식 네트워크 (Knowledge Network)
- **대상**: Entry 간의 `link` 및 `baseline` 관계.
- **표현**: Directed Acyclic Graph (DAG) 기반의 노드-엣지 다이어그램.
- **특징**: 
    - `baseline`은 부모-자식 관계(계보)로, `link`는 관련성(네트워크)으로 구분하여 렌더링.
    - 태그별로 노드 색상을 구분하여 관심 영역 시각화.

### B. 계보 타임라인 (Genealogy Timeline)
- **대상**: 특정 Entry의 `revision chain`과 파생 Entry들.
- **표현**: 가로축(시간) 기반의 브랜칭 다이어그램 (Git Graph와 유사).
- **특징**: 
    - (Proposal 005 연계) 작성자(`author`)별로 브랜치를 색칠하여 누가 어떤 변경을 가했는지 표시.
    - 논쟁이 발생한 지점(동일 baseline에서의 다중 delta)을 갈래길로 표시.

### C. 기여도 및 밀도 (Contribution & Density)
- **대상**: Tag, Author, Timestamp.
- **표현**: 트리맵(Treemap) 또는 버블 차트(Bubble Chart).
- **특징**: 
    - 특정 태그에 포함된 Entry가 많을수록 영역이 크게 표시됨.
    - 팀의 지식 자산이 어디에 집중되어 있는지 "지식 분포도" 제공.

---

## 3. 기술적 구현 제안

시각화 엔진을 CLI 내부에 직접 구현하기보다, 범용 포맷으로 출력하여 외부 도구와 연계하는 방식을 제안합니다.

1.  **`elf graph --format dot`**: Graphviz 등에서 렌더링 가능한 DOT 파일 출력.
2.  **`elf graph --format json`**: D3.js, Cytoscape.js 등 웹 기반 시각화 라이브러리에 입력할 수 있는 구조화된 데이터 제공.
3.  **`elf graph --ui` (추후 고려)**: 로컬 호스트에서 간단한 웹 서버를 띄워 인터랙티브한 그래프(예: Force-directed graph)를 보여주는 기능.

## 4. 시각화가 주는 팀의 가치

1.  **지식의 병목 발견**: 너무 많은 링크가 걸려 있는 Entry는 과부하된 주제일 수 있음을 시각적으로 경고.
2.  **의사결정 히스토리 탐색**: "왜 이 프로젝트가 이렇게 변했지?"라는 질문에 대해 타임라인을 따라가며 직관적으로 이해.
3.  **지식 공유의 투명성**: 시각화된 지식 지도를 통해 팀원 누구나 프로젝트의 현재 상태와 히스토리를 동일하게 파악.
